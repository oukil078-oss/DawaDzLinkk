Étape 1 : Connexion au VPS

1 Ouvrir PuTTY et se connecter

Sur Windows :

  1. Ouvrez PuTTY
  2. Dans Host Name, entrez : VOTRE_IP_SERVEUR
  3. Port : 22
  4. Cliquez Open
  5. Login : root
  6. Entrez votre mot de passe (il ne s'affiche pas, c'est normal)

Sur Mac/Linux :

ssh root@VOTRE_IP_SERVEUR
✅ Connexion réussie si vous voyez :
root@ubuntu:~#

Étape 2 : Installation de Dokploy

2 Une seule commande à exécuter

🎯 LA commande magique

Copiez et collez cette commande, puis appuyez sur Entrée :

curl -sSL https://dokploy.com/install.sh | sh
⏳ Attendez environ 2-3 minutes...

L'installation va :

  • Installer Docker automatiquement
  • Configurer Dokploy
  • Démarrer tous les services
✅ Installation terminée quand vous voyez :
Congratulations! Dokploy is now installed.
Access the panel at: http://VOTRE_IP:3000

Étape 3 : Premier Accès au Dashboard

3 Accéder à l'interface Dokploy

3.1 Ouvrir le navigateur

Dans votre navigateur web, allez à :

http://VOTRE_IP_SERVEUR:3000

Exemple : http://192.168.1.100:3000

3.2 Créer le compte administrateur

Remplissez le formulaire :

Champ Valeur
Email votre@email.com
Password Un mot de passe fort (min. 8 caractères)
⚠️ Important

Notez bien ces identifiants ! Vous en aurez besoin pour accéder au dashboard.

✅ Vous êtes maintenant sur le Dashboard Dokploy !

Étape 4 : Création du Projet

4 Créer un nouveau projet DawaLink

Navigation dans le Dashboard :

Projects → + Create Project

Remplir les informations :

Champ Valeur
Project Name dawalink
Description Marketplace B2B Pharmaceutique

Cliquez sur Create

✅ Projet créé !

Vous allez maintenant voir le projet "dawalink" dans la liste.

Étape 5 : Création de la Base MongoDB

5 Ajouter MongoDB au projet

5.1 Ouvrir le projet

Cliquez sur le projet dawalink

5.2 Ajouter une base de données

+ Create Service → Database → MongoDB

5.3 Configurer MongoDB

Champ Valeur
Name dawalink-db
Database Name dawalink
Database User dawalink_user
Database Password VotreMotDePasseSecurise123!
📝 NOTEZ CES INFORMATIONS !

Vous aurez besoin de ces identifiants pour connecter le backend.

Cliquez sur Create

5.4 Récupérer l'URL de connexion MongoDB

Après création, cliquez sur dawalink-db, puis allez dans l'onglet Environment.

Vous verrez une URL comme :

mongodb://dawalink_user:VotreMotDePasseSecurise123!@dawalink-db:27017/dawalink
📋 Copiez cette URL !

Elle sera utilisée à l'étape suivante.

Étape 6 : Déploiement du Backend (FastAPI)

6 Déployer l'API Python

6.1 Créer le service Backend

+ Create Service → Application

6.2 Configurer le service

Champ Valeur
Name dawalink-backend
Source Git (ou Upload si pas de GitHub)

6.3 Si vous utilisez GitHub :

Provider → GitHub → Connect

Sélectionnez votre repository et la branche main

Dans Build Path, entrez : /backend

6.4 Configurer les Variables d'Environnement

Onglet Environment

Ajoutez ces variables :

MONGO_URL=mongodb://dawalink_user:VotreMotDePasseSecurise123!@dawalink-db:27017/dawalink
DB_NAME=dawalink
JWT_SECRET=votre_cle_secrete_tres_longue_et_aleatoire_minimum_32_caracteres
💡 Générer une clé JWT secrète

Allez sur : https://generate-secret.vercel.app/32

6.5 Configurer le Port

Onglet Advanced → Ports
Port interne Port exposé
8001 8001

6.6 Déployer

Cliquez sur Deploy
✅ Backend déployé quand le statut est : Running

Étape 7 : Déploiement du Frontend (React)

7 Déployer l'interface utilisateur

7.1 Créer le service Frontend

+ Create Service → Application

7.2 Configurer le service

Champ Valeur
Name dawalink-frontend
Source Git
Build Path / (racine)

7.3 Configurer les Variables d'Environnement

Onglet Environment 
VITE_API_URL=https://api.dawalink.dz

(Remplacez par votre domaine API ou l'IP du serveur)

7.4 Configurer le Build

Onglet Advanced → Build
Champ Valeur
Build Command yarn install && yarn build
Output Directory build

7.5 Configurer le Port

Port interne Port exposé
80 80

7.6 Déployer

Cliquez sur Deploy
✅ Frontend déployé !

Étape 8 : Configuration du Domaine

8 Connecter votre nom de domaine

8.1 Chez votre registrar de domaine

Connectez-vous à votre compte (ex: GoDaddy, Namecheap, OVH) et configurez les DNS :

Type Nom Valeur TTL
A @ VOTRE_IP_SERVEUR 3600
A www VOTRE_IP_SERVEUR 3600
A api VOTRE_IP_SERVEUR 3600

8.2 Dans Dokploy - Backend

dawalink-backend → Domains → Add Domain
Champ Valeur
Domain api.dawalink.dz

8.3 Dans Dokploy - Frontend

dawalink-frontend → Domains → Add Domain
Champ Valeur
Domain dawalink.dz
Domain 2 www.dawalink.dz
⏳ Propagation DNS

Les changements DNS peuvent prendre jusqu'à 24-48h pour se propager.

Étape 9 : Activation HTTPS (SSL)

9 Sécuriser avec un certificat SSL gratuit

Pour chaque service (Backend et Frontend) :

Service → Domains → Votre domaine → Generate SSL

Dokploy génère automatiquement un certificat Let's Encrypt gratuit !

✅ HTTPS activé !

Votre site est maintenant accessible en https://

🔄 Renouvellement automatique

Dokploy renouvelle automatiquement les certificats avant expiration.

📚 Guide de Maintenance

Commandes Utiles dans Dokploy

Action Comment faire
Voir les logs Service → Logs
Redémarrer un service Service → Actions → Restart
Mettre à jour le code Service → Deploy (après push sur GitHub)
Voir les statistiques Service → Monitoring
Sauvegarder la BDD dawalink-db → Backups → Create Backup

🔄 Déploiement Automatique (CI/CD)

Activer le déploiement automatique :
Service → Settings → Auto Deploy → Enable

Chaque git push déclenchera automatiquement un nouveau déploiement !

🔧 Résolution de Problèmes

Problème Solution
Service ne démarre pas Vérifiez les logs (Service → Logs)
Erreur de connexion MongoDB Vérifiez l'URL dans les variables d'environnement
Site non accessible Vérifiez que les DNS sont bien configurés
Erreur 502 Bad Gateway Le backend n'est pas démarré - redémarrez-le
SSL ne fonctionne pas Attendez la propagation DNS (24-48h)
