1. Prérequis

📦 Ce dont vous avez besoin :
  • VPS Octenium Advanced activé avec Ubuntu 22.04
  • Adresse IP de votre serveur (fournie par Octenium)
  • Mot de passe root (fourni par email)
  • Nom de domaine (ex: dawalink.dz) - optionnel au début
  • Terminal SSH : PuTTY (Windows) ou Terminal (Mac/Linux)

💻 Installer PuTTY (Windows)

Téléchargez PuTTY depuis : https://www.putty.org

2. Connexion au VPS

Étape 2.1 : Se connecter en SSH

Avec PuTTY (Windows) :

  1. Ouvrez PuTTY
  2. Dans "Host Name", entrez l'IP de votre serveur
  3. Port : 22
  4. Cliquez "Open"
  5. Login : root
  6. Entrez votre mot de passe

Avec Terminal (Mac/Linux) :

ssh root@VOTRE_IP_SERVEUR
✅ Connexion réussie si vous voyez :
root@serveur:~#

3. Sécurisation du Serveur

Étape 3.1 : Mettre à jour le système

# Mettre à jour la liste des paquets
apt update

# Installer les mises à jour
apt upgrade -y

Étape 3.2 : Créer un utilisateur (recommandé)

# Créer un nouvel utilisateur
adduser dawalink

# Lui donner les droits sudo
usermod -aG sudo dawalink

Étape 3.3 : Configurer le pare-feu

# Installer et configurer UFW
apt install ufw -y

# Autoriser SSH, HTTP, HTTPS
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp

# Activer le pare-feu
ufw enable

# Vérifier le statut
ufw status

4. Installation des Logiciels

Étape 4.1 : Installer Node.js 20

# Ajouter le repository NodeSource
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo bash -

# Installer Node.js
apt install nodejs -y

# Vérifier l'installation
node --version   # Doit afficher v20.x.x
npm --version    # Doit afficher 10.x.x

Étape 4.2 : Installer Yarn

npm install -g yarn

# Vérifier
yarn --version

Étape 4.3 : Installer Python 3.11

# Python est généralement préinstallé sur Ubuntu 22.04
python3 --version

# Installer pip et venv
apt install python3-pip python3-venv -y

Étape 4.4 : Installer MongoDB 7.0

# Importer la clé GPG de MongoDB
curl -fsSL https://www.mongodb.org/static/pgp/server-7.0.asc | \
   gpg -o /usr/share/keyrings/mongodb-server-7.0.gpg --dearmor

# Ajouter le repository
echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-7.0.gpg ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | tee /etc/apt/sources.list.d/mongodb-org-7.0.list

# Mettre à jour et installer
apt update
apt install mongodb-org -y

# Démarrer MongoDB
systemctl start mongod
systemctl enable mongod

# Vérifier le statut
systemctl status mongod

Étape 4.5 : Installer Nginx

apt install nginx -y

# Démarrer Nginx
systemctl start nginx
systemctl enable nginx

Étape 4.6 : Installer PM2 (Gestionnaire de processus)

npm install -g pm2

Étape 4.7 : Installer Git

apt install git -y

5. Configuration MongoDB

Étape 5.1 : Créer un utilisateur admin

# Se connecter à MongoDB
mongosh

# Dans le shell MongoDB :
use admin
db.createUser({
  user: "dawalinkAdmin",
  pwd: "VOTRE_MOT_DE_PASSE_SECURISE",
  roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]
})

# Créer la base de données DawaLink
use dawalink
db.createUser({
  user: "dawalinkUser",
  pwd: "AUTRE_MOT_DE_PASSE_SECURISE",
  roles: [ { role: "readWrite", db: "dawalink" } ]
})

# Quitter
exit
⚠️ Important

Notez bien les mots de passe ! Vous en aurez besoin pour la configuration.

6. Déploiement du Projet

Étape 6.1 : Créer le dossier du projet

mkdir -p /var/www/dawalink
cd /var/www/dawalink

Étape 6.2 : Transférer les fichiers

Option A : Via Git (si le code est sur GitHub)

git clone https://github.com/VOTRE_USERNAME/dawalink.git .

Option B : Via SFTP/SCP

Utilisez FileZilla ou WinSCP pour transférer les fichiers vers /var/www/dawalink

Étape 6.3 : Installer les dépendances Frontend

cd /var/www/dawalink
yarn install

Étape 6.4 : Construire le Frontend

yarn build

Étape 6.5 : Configurer le Backend

# Créer l'environnement virtuel Python
cd /var/www/dawalink/backend
python3 -m venv venv

# Activer l'environnement
source venv/bin/activate

# Installer les dépendances
pip install -r requirements.txt

Étape 6.6 : Créer le fichier .env Backend

nano /var/www/dawalink/backend/.env

Contenu du fichier :

MONGO_URL=mongodb://dawalinkUser:VOTRE_MOT_DE_PASSE@localhost:27017/dawalink
DB_NAME=dawalink
JWT_SECRET=GENEREZ_UNE_CLE_SECRETE_LONGUE_ET_ALEATOIRE
💡 Générer une clé secrète
openssl rand -hex 32

7. Configuration Nginx

Étape 7.1 : Créer la configuration DawaLink

nano /etc/nginx/sites-available/dawalink

Contenu du fichier :

server {
    listen 80;
    server_name dawalink.dz www.dawalink.dz;  # Remplacez par votre domaine

    # Frontend (fichiers statiques)
    location / {
        root /var/www/dawalink/build;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API Backend
    location /api {
        proxy_pass http://127.0.0.1:8001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}

Étape 7.2 : Activer la configuration

# Créer le lien symbolique
ln -s /etc/nginx/sites-available/dawalink /etc/nginx/sites-enabled/

# Supprimer la config par défaut
rm /etc/nginx/sites-enabled/default

# Tester la configuration
nginx -t

# Redémarrer Nginx
systemctl restart nginx

8. Certificat SSL (HTTPS)

Étape 8.1 : Installer Certbot

apt install certbot python3-certbot-nginx -y

Étape 8.2 : Obtenir le certificat

certbot --nginx -d dawalink.dz -d www.dawalink.dz

Suivez les instructions et entrez votre email.

Étape 8.3 : Renouvellement automatique

# Tester le renouvellement
certbot renew --dry-run
✅ Le certificat se renouvellera automatiquement !

9. Gestion des Processus (PM2)

Étape 9.1 : Créer le fichier de configuration PM2

nano /var/www/dawalink/ecosystem.config.js

Contenu :

module.exports = {
  apps: [
    {
      name: 'dawalink-backend',
      cwd: '/var/www/dawalink/backend',
      script: 'venv/bin/uvicorn',
      args: 'server:app --host 0.0.0.0 --port 8001',
      interpreter: 'none',
      env: {
        NODE_ENV: 'production'
      }
    }
  ]
};

Étape 9.2 : Démarrer l'application

cd /var/www/dawalink
pm2 start ecosystem.config.js

# Vérifier le statut
pm2 status

# Voir les logs
pm2 logs

Étape 9.3 : Configurer le démarrage automatique

# Sauvegarder la configuration
pm2 save

# Configurer le démarrage au boot
pm2 startup

12. Maintenance

Commandes utiles :

Action Commande
Voir les logs backend pm2 logs dawalink-backend
Redémarrer le backend pm2 restart dawalink-backend
Voir les logs Nginx tail -f /var/log/nginx/error.log
Redémarrer Nginx systemctl restart nginx
Redémarrer MongoDB systemctl restart mongod
Mettre à jour le code cd /var/www/dawalink && git pull && yarn build && pm2 restart all
Sauvegarder la BDD mongodump --db dawalink --out /backup/

Sauvegardes automatiques :

# Créer un script de sauvegarde
nano /usr/local/bin/backup-dawalink.sh

Contenu :

#!/bin/bash
DATE=$(date +%Y%m%d)
mongodump --db dawalink --out /backup/dawalink-$DATE
find /backup -mtime +7 -delete  # Supprimer les backups > 7 jours
# Rendre exécutable et planifier
chmod +x /usr/local/bin/backup-dawalink.sh
crontab -e

# Ajouter cette ligne pour une sauvegarde quotidienne à 3h du matin
0 3 * * * /usr/local/bin/backup-dawalink.sh
